Section 4 of 4 — How everything connects end-to-end

🖥️ Express Backend API (localhost:3001)
POST /upload/:cameraId
ESP32-CAM sends JPEG → runs Gemma 4 → returns guidance
GET /stream/:cameraId
SSE stream → pushes guidance to portal in real-time
GET /cameras
List all registered cameras + online status
POST /cameras/register
Register new camera with name + location
GET /patients/:id/performance
Returns analytics + AI weekly summary
GET /alerts?patientId=
Paginated alert log with photos + AI reasoning
🤖 Gemma 4 Detection Prompts
CONFUSION / WANDERING
"Analyse this image of a dementia patient. Is the person standing still with a confused expression, wandering without purpose, or in an unfamiliar location? Return JSON: {detected: bool, severity: low|medium|critical, guidance: string}"
FALL DETECTION
"Is the person in this image fallen, on the floor, or in an unsafe posture? Return JSON with severity and immediate guidance text."
FACE RECOGNITION
"Are there multiple people visible? Describe the interaction. Is the patient calm or distressed with this person present?"
ROUTINE CHECK
"Based on time {time} and patient routine {routine}, is the patient doing the right activity? Provide gentle next-step guidance."
📋 Full Setup Guide — Step by Step
STEP 1 — Local Machine
# Install Ollama
brew install ollama
ollama pull gemma4:12b-it

# Install Cloudflare Tunnel
brew install cloudflared
cloudflared tunnel login
cloudflared tunnel create staysync
cloudflared tunnel route dns \
 staysync staysync.yourdomain.com

# Start tunnel
cloudflared tunnel run staysync
STEP 2 — Backend + Vercel
# Clone & install
cd staysync
npm install
node server.js # port 3001

# Deploy frontend
npx create-next-app portal
cd portal
# set NEXT_PUBLIC_API_URL=
# https://staysync.yourdomain.com
vercel deploy
