Proposed Architecture Approaches

Three ways to build the StaySync dementia care system

A

Next.js + Node.js (Recommended)

Frontend: Next.js on Vercel  |  Backend: Express + Ollama (local)

  • Next.js deploys perfectly to Vercel, built-in API routes
  • Your existing Express project becomes the local backend
  • Ollama serves Gemma 4 vision locally via HTTP
  • Cloudflare Tunnel exposes local backend to Vercel
  • Web Serial API handles USB camera setup in browser
  • Browser TTS (Web Speech API) for audio guidance
Least context switching — stays in JS
B

Next.js + Python FastAPI

Frontend: Next.js on Vercel  |  Backend: FastAPI + Ollama (local)

  • Python backend is more natural for AI/ML workflows
  • FastAPI has async support for concurrent camera uploads
  • Better ecosystem if you add more AI features later
  • Requires running two separate local processes
Best if you know Python
C

Fully Local (No Vercel)

Frontend + Backend: All on your Mac (Express serves React)

  • Simpler — one process, no tunnel needed
  • Access from phone via local IP (same WiFi required)
  • No internet dependency at all
  • Phone must be on same WiFi as Mac always
  • No Vercel URL to share, harder to demo for Kaggle
Limited for Kaggle demo
