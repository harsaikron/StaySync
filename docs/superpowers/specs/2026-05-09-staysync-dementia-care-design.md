# StaySync — Dementia Care Portal Design Spec
**Date:** 2026-05-09  
**Competition:** Kaggle Gemma 4 for Good Hackathon  
**Stack:** Next.js (Vercel) + Express (local) + Ollama (Gemma 4) + ESP32-CAM

---

## 1. Problem Statement

Dementia patients require continuous monitoring and real-time guidance to safely carry out daily routines. Caregivers cannot always be physically present. StaySync uses ESP32-CAM hardware and a locally-running Gemma 4 vision model to detect patient situations and deliver spoken guidance to the patient's mobile phone — with no cloud AI costs and no third-party data sharing.

---

## 2. System Architecture

```
ESP32-CAM  ──POST /upload/:cameraId──▶  Express Backend (localhost:3001)
                                                │
                                         calls Ollama HTTP API
                                                │
                                        Gemma 4 (gemma4:12b-it)
                                        runs locally via Ollama
                                                │
                                         guidance JSON returned
                                                │
                              ┌─────────────────▼──────────────────┐
                              │         Cloudflare Tunnel           │
                              │  https://staysync.yourdomain.com   │
                              │      ←→  localhost:3001             │
                              └─────────────────┬──────────────────┘
                                                │ SSE stream
                                        Next.js Portal (Vercel)
                                                │
                                         Patient's Phone (Chrome)
                                         Web Speech API → speaker
```

### Key Components

| Component | Technology | Where it runs |
|-----------|-----------|--------------|
| Camera hardware | ESP32-CAM | Physical device |
| Backend API | Node.js + Express | Local Mac (port 3001) |
| AI model | Gemma 4 12B-IT via Ollama | Local Mac (port 11434) |
| Public tunnel | Cloudflare Tunnel | Mac → public HTTPS URL |
| Web portal | Next.js | Vercel (free tier) |
| Audio guidance | Web Speech API (TTS) | Patient's phone browser |
| Camera setup | Web Serial API | Chrome on Mac (one-time) |

## 3. Portal Pages

### 3.1 `/dashboard` — Caregiver Home
- Live camera feed grid (all registered cameras, JPEG snapshots refreshed every 3s)
- Active alert banner (fall / wandering detected) pinned at top
- Today's risk score bar (0–100, AI-generated)
- Latest Gemma 4 guidance per camera
- "Speak now" button + auto-speak toggle
- Bottom tab navigation: Home / Cameras / Stats / Patient

### 3.2 `/patient` — Patient-Facing Screen (simplified)
- Large text display of current guidance ("Time to take your morning medicine 💊")
- Auto-speaks via Web Speech API when new guidance arrives via SSE
- Large "Repeat" button to replay last spoken message
- Calming dark UI, minimal distractions
- Installable as PWA — patient adds to iPhone/Android home screen, opens full-screen

### 3.3 `/cameras` — Camera Management
- List all registered cameras with online/offline status and last-seen timestamp
- Link to individual live view per camera
- Remove camera option

### 3.4 `/cameras/setup` — ESP32-CAM WiFi Setup (key feature)
- "Connect Camera" button triggers Web Serial API (Chrome only)
- Detects ESP32-CAM on USB automatically
- User enters: camera name, room location, home WiFi SSID, WiFi password
- One-click flashes credentials to device over serial
- Camera registers itself with backend on first WiFi connection
- Requires Chrome browser on the Mac the camera is plugged into

### 3.5 `/patients/:id` — Patient Profile
- Name, age, photo, emergency contacts
- Diagnosed conditions and medications
- Daily routine schedule (wake, meals, medications, sleep times)
- Assigned cameras (which rooms monitor this patient)
- Caregiver notes feed
- Current AI risk level badge: Low / Medium / High

### 3.6 `/patients/:id/performance` — AI Analytics Dashboard
- Daily confusion frequency chart (7-day and 30-day views)
- Wandering incidents heatmap by time of day
- Medicine adherence percentage (detected vs scheduled)
- Fall risk trend score (AI-generated, 0–100)
- Sleep pattern indicator (night movement detection)
- Weekly AI narrative summary generated by Gemma 4
- AI suggestions panel (real-time, daily, weekly — see Section 5)

### 3.7 `/patients/:id/timeline` — Behavior Timeline
- Chronological event log with photo thumbnails
- Filter by: alerts / normal / all events
- Each event shows AI reasoning text
- Exportable PDF for doctor visits

### 3.8 `/alerts` — Alert Log
- Full history of Gemma 4 detections
- Severity badge: Info / Warning / Critical
- Thumbnail of triggering photo
- Guidance text that was spoken
- Timestamp and camera name
- AI confidence score per detection
- "I'm handling it" one-tap dismiss

## 4. Backend API

All routes served by Express on `localhost:3001`, exposed publicly via Cloudflare Tunnel.

| Method | Route | Description |
|--------|-------|-------------|
| POST | `/upload/:cameraId` | Receives JPEG from ESP32-CAM, runs Gemma 4, stores result, pushes to SSE |
| GET | `/stream/:cameraId` | SSE endpoint — pushes guidance events to portal in real-time |
| GET | `/cameras` | Returns list of all registered cameras with status |
| POST | `/cameras/register` | Registers new camera (name, location, id) |
| DELETE | `/cameras/:id` | Removes a camera |
| GET | `/patients` | List all patients |
| POST | `/patients` | Create patient profile |
| GET | `/patients/:id` | Get patient details |
| PUT | `/patients/:id` | Update patient profile |
| GET | `/patients/:id/performance` | Returns analytics data + AI weekly summary |
| GET | `/alerts` | Paginated alert log, filterable by patientId / severity |
| GET | `/patients/:id/timeline` | Chronological event log for timeline view |
| POST | `/reports/generate/:patientId` | Manually trigger daily/weekly AI summary generation |

### Scheduled Jobs (node-cron)
- **Daily at 11pm** — runs Gemma 4 over the day's event log, writes summary to `reports` table
- **Sunday at 8pm** — runs Gemma 4 over 7-day event log, writes weekly narrative + suggestions

### Data Storage
- SQLite (via `better-sqlite3`) — zero-config, single file, no separate DB process
- Tables: `cameras`, `patients`, `events`, `alerts`, `reports`
- Photos stored as files in `uploads/` directory, path saved in DB

## 5. Gemma 4 Integration

Model: `gemma4:12b-it` pulled via Ollama. Called via `http://localhost:11434/api/generate` with base64-encoded image.

### 5.1 Detection Prompts

**Confusion / Wandering:**
```
You are a dementia care AI assistant. Analyse this image of a patient.
Detect if the person appears confused (blank stare, disorientation), 
is wandering without purpose, or is in an unexpected location given 
the time {time} and routine {routine_summary}.
Return ONLY valid JSON: {"detected": bool, "type": "confusion|wandering|normal", 
"severity": "low|medium|critical", "guidance": "short spoken sentence for patient", 
"reasoning": "one sentence explanation"}
```

**Fall Detection:**
```
Analyse this image for fall or unsafe posture.
Is the person on the floor, fallen, or in a dangerous position?
Return ONLY valid JSON: {"detected": bool, "severity": "low|medium|critical",
"guidance": "short spoken sentence", "reasoning": "one sentence"}
```

**Face / Visitor Recognition:**
```
Are there multiple people visible in this image? Describe the interaction.
Is the patient calm or distressed with this person present?
Return ONLY valid JSON: {"multiple_people": bool, "patient_state": "calm|distressed|neutral",
"guidance": "short spoken sentence", "reasoning": "one sentence"}
```

**Routine Check:**
```
Current time is {time}. The patient's schedule is: {routine_json}.
Based on this image, is the patient doing the correct activity for this time?
Return ONLY valid JSON: {"on_schedule": bool, "current_activity": "string",
"expected_activity": "string", "guidance": "short spoken sentence", "reasoning": "one sentence"}
```

**Medicine / Meal Reminder:**
```
The time is {time}. The patient is due to {action} at {scheduled_time}.
Look at this image and determine if the patient has already done this.
Return ONLY valid JSON: {"completed": bool, "guidance": "short spoken sentence",
"reasoning": "one sentence"}
```

### 5.2 AI Suggestions Engine — Three Tiers

| Tier | Trigger | Output |
|------|---------|--------|
| Real-time | Every photo upload | Spoken guidance via SSE → Web Speech API |
| Daily report | Cron at 11pm | Good/bad day score, most confused time, routine completion %, suggestions |
| Weekly insights | Cron Sunday 8pm | Behavior trend (improving/declining), caregiver schedule recommendations, flag for doctor |

### 5.3 Performance Analytics
- Confusion frequency: count of `type=confusion` events per hour, aggregated by day
- Wandering incidents: count of `type=wandering` events, heatmap by hour-of-day
- Medicine adherence: scheduled doses vs `completed=true` detections
- Fall risk score: weighted formula — (falls×40 + critical_events×20 + confusion_rate×10) / 7-day window, capped at 100
- Weekly AI summary: Gemma 4 receives the 7-day event log as text and generates a narrative

## 6. Mobile UX Design Principles

- **Dark theme** throughout — easy on eyes at night, accessible for elderly users
- **Bottom tab navigation** — thumb-reachable on any phone size
- **Minimum 48px tap targets** — accessibility standard for motor-impaired users
- **PWA** — patient installs portal to home screen via "Add to Home Screen" in Chrome/Safari; opens full-screen without browser chrome
- **Auto-speak mode** — SSE guidance triggers `window.speechSynthesis.speak()` automatically; patient never needs to tap
- **Critical alerts always visible** — red banner pinned to top of dashboard, not dismissible without explicit tap
- **Large text on patient screen** — minimum 24px body text, guidance text 28px+

## 7. ESP32-CAM Firmware Behaviour

- On boot: connects to configured WiFi using stored credentials
- Captures JPEG every 3 seconds (configurable via serial before flash)
- Posts JPEG to `https://staysync.yourdomain.com/upload/:cameraId` as `multipart/form-data`
- If upload fails (no WiFi / tunnel down): retries after 10 seconds, stores nothing locally
- Camera ID is the ESP32 chip ID (unique, set during serial setup)
- Arduino IDE sketch provided in `firmware/esp32cam.ino`

## 8. Step-by-Step Connection Guide

### Step 1 — Install Ollama and pull Gemma 4 (Mac)
```bash
brew install ollama
ollama serve                        # starts on localhost:11434
ollama pull gemma4:12b-it           # ~8GB download, one-time
```

### Step 2 — Set up Cloudflare Tunnel (Mac)
```bash
brew install cloudflared
cloudflared tunnel login            # opens browser, sign in to Cloudflare
cloudflared tunnel create staysync
# Add DNS route in Cloudflare dashboard or:
cloudflared tunnel route dns staysync staysync.yourdomain.com
# Start tunnel (keep running):
cloudflared tunnel run staysync
```
> Free Cloudflare account + domain required. Alternatively use `cloudflared tunnel --url localhost:3001` for a temporary trycloudflare.com URL (no account needed, good for demo).

### Step 3 — Run the local backend
```bash
cd staysync
npm install
TUNNEL_URL=https://staysync.yourdomain.com node server.js
# Runs on localhost:3001
```

### Step 4 — Deploy the portal to Vercel
```bash
cd portal                           # Next.js app
# Set environment variable in Vercel dashboard:
# NEXT_PUBLIC_API_URL = https://staysync.yourdomain.com
vercel deploy --prod
# Portal is now live at https://staysync-xxx.vercel.app
```

### Step 5 — Flash ESP32-CAM (one-time per camera)
1. Plug ESP32-CAM into Mac via USB-to-TTL adapter
2. Open portal in **Chrome** on Mac → go to `/cameras/setup`
3. Click **"Connect Camera"** → browser prompts for serial port access → select the ESP32 port
4. Enter camera name, room, WiFi SSID, WiFi password
5. Click **"Flash & Connect"** — credentials sent over serial, camera reboots
6. Camera appears as online in `/cameras` within 10 seconds

### Step 6 — Patient phone setup
1. Open Vercel portal URL in **Chrome** (Android) or **Safari** (iPhone)
2. Tap **Share → Add to Home Screen** (iPhone) or **Install App** (Android)
3. Open the installed app → go to `/patient`
4. Tap anywhere to allow audio — browser requires one user gesture for TTS
5. Leave screen on — guidance will auto-speak whenever Gemma 4 detects a situation

## 10. Out of Scope (v1)

- Multi-patient multi-caregiver role management (single caregiver per install)
- Cloud backup of photos or events
- Native iOS/Android app
- Video streaming (JPEG snapshots only — lower bandwidth, easier on ESP32-CAM)
- Face recognition with identity matching (Gemma 4 describes people, does not identify by name)
